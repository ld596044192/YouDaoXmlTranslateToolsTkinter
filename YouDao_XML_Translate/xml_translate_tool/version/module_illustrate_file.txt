欢迎使用 xml批量翻译工具，部分功能可能需要进行说明才能正确使用，否则会造成意想不到的后果，请注意阅读以下模块说明

（1）批量翻译功能 - （启动工具后主界面）
功能：主要根据模板进行批量翻译，使用的引擎目前暂时为 有道翻译
使用说明：
1.翻译文件夹路径：
此路径为模板路径，例如res文件夹下，必须有values\strings.xml（此strings.xml文件中的文本内容必须是没有进行翻译的，即中文）；
在values路径下必须至少有一个文件夹是values-langID（此langID是翻译语言代号，如英文就是en，德语就是de）；
因为程序会通过识别langID确定你接下来要翻译的语言，因此langID必须是正确的，且是有道翻译支持的langID；
想要批量翻译多少种语言，那么就需要创建多个values-langID文件夹即可！
2.自定义翻译（如果文本翻译框有内容时，就会只翻译自定义翻译文本框中的内容，不会按照上方项目中的values为基准去翻译）
可以自定义输入需要翻译的内容，此内容必须为这种xml格式： <string name="app_name">自定义翻译</string>
如果含有注释内容，必须为xml格式 <!--这是注释-->，不会被翻译，且追加到文件的翻译内容的前面和后面，具体效果自己尝试！！！

------------------------------------------
如看不懂上面的，那么点击右侧的“模板”按钮，会自动新建一个模板（桌面上新生成xml_translate_template文件夹），里面含有values和values-en文件夹，你只需要把values里面的strings.xml更换为你项目中values文件夹中的strings.xml（中文文本内容），然后把values-en文件夹后面的en换成你想要翻译的语言代号，批量翻译多个就复制多个values-en文件夹后重命名即可！
2.点击翻译：开始批量翻译，实时显示翻译进度，可以点击“复制”按钮复制当前翻译的内容，粘贴到你想要粘贴的地方
3.停止翻译：如果翻译过程中翻译错误或者操作失误，可以点击“停止翻译”，这样就不会再继续翻译了
4.暂停翻译/恢复翻译：在翻译过程中可以暂停翻译，也可以恢复翻译，适用于需要实时监视翻译过程的情况，操作灵活！

（2）单个xml翻译 - 与批量翻译功能相比，就是根据单个xml进行翻译
功能：根据用户提供的xml文件（xml格式要正确，带有name值和文本内容）进行翻译
使用说明：
1.单个xml文件路径：选择单个xml文件，该xml文件必须符合xml格式，要有根节点和name属性名称和值，还有文本内容
2.翻译语言：目前使用的是有道翻译引擎，因此支持的语言是有道翻译引擎支持，选择翻译语言对应就是翻译哪种语言
3.点击翻译、停止翻译、暂停/恢复翻译按钮：功能与（1）批量翻译功能中的一致
4.打开翻译文件夹：点击直接打开文件夹或者手动打开桌面上的single_translate_da文件夹

（3）xml合并工具 - 定制功能
功能：把第三方翻译的xml文件（多国语言）合并到项目中
实现方式：
经彭元元要求，客户或第三方翻译出来的内容转为多国语言xml文件，然后需要跟项目中的xml进行对比；
要求项目中的模块文件夹中存在以下层级路径：模块文件夹\src\main\res\values,没有的就不能使用该功能（算是定制功能）；
多国语言xml文件与模块文件夹\src\main\res\values\strings.xml文件进行对比，如果两者的name值为同一个，则把多国语言的xml文件中的同一个name值所在的一整行内容提取出来，合并到用户设置的文件夹中的strings.xml中！
简单的说，就是把第三方或客户翻译的内容合并到项目中，这样就不需要自己手动去找到对应的翻译并复制粘贴到项目，程序帮你对比并合并到项目中去了！！！
使用说明：
1.多国语言xml文件路径：客户提供或第三方翻译的文本经转换的xml文件
2.项目路径：就是你要操作的项目路径，比如res文件夹路径，下一层就是各个模块的文件夹
3.合并后的文件夹名称：默认已有values-，只要在-后面填写需要翻译的语言代号即可，如en；注意，该语言代号必须与多国语言xml文件中翻译的语言一致，比如多国语言xml中的文本内容都是英文en翻译出来的，那么需要填写en；填写错误则合并的内容也就错误，所以要求一致！
4.合并按钮：点击“合并”按钮，程序就会开始进行合并！！！
还有就是如果合并后的文件夹名称存在的话，就会跳过不会去翻译，同时也在消息日志框中记录对应的路径，方便用户自己去找！

（4）xml对比工具 - 定制功能
功能：通过对比两个xml文件，找到name值相同但文本内容不同，然后保存在新的xml文件中
实现方式：
经彭元元要求，客户或第三方翻译出来的内容转为多国语言xml文件，通过跟需要对比的xml文件进行对比，找到name值相同但文本内容不同，然后保存在新的xml文件中；
主要是用于客户或第三方翻译出来的内容与项目已有的内容进行对比，从而更新项目中的内容为客户或第三方翻译出来的内容！
1.多国语言xml文件路径：客户提供或第三方翻译的文本经转换的xml文件
2.目标xml路径：项目中的strings.xml，建议里面的内容翻译语言与多国语言xml文件中的一致，否则会全部被更新！！！
3.Xml文件合并按钮：点击该按钮，程序就会自动进行比较；
比较完成后，会自动生成新的xml文件在桌面，名称为new_xml_da.xml
